# Testing Best Practices (Human Reference)

This directory contains verbose, educational documentation on test derivation and quality — targeting human developers who need to understand V-model testing expectations.

These documents are the unabridged versions of the corresponding agentic skills in `.claude/skills/`. The skills are optimized for LLM consumption (concise, no redundancy). These docs are optimized for human learning (examples, explanations, rationale).

## Contents

- **test-derivation-skill.md** — Full verbose version of the derive-test-cases skill, including the 5-step process, detailed sub-instructions, and handoff checklist
- **derivation-strategies.md** — Four V-model test derivation strategies with detailed examples, tables, and edge case guidance
- **testing-anti-patterns.md** — Nine testing anti-patterns with code examples, explanations, and fixes
